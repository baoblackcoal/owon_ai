OWON AI 助手

Phase 1: MVP - UX 原型设计展示 已更新

项目概述

本项目旨在为 OWON 仪器仪表用户构建一个智能化的 AI 助手网页应用。MVP 阶段专注于核心功能：文字问答、多对话管理、人工支持工作流和增强版管理后台。

Phase 1 功能范围：

  • 仅限文字的AI问答，支持流式输出和Markdown渲染
  • 对话历史记录（仅限注册用户）
  • AI回答的点赞/点踩功能
  • 基础的管理员后台：能查看所有用户和问答流（点赞，点踩的过滤）
  • 上线人工支持工作流（请求、后台处理、邮件通知）

重要更新：将问答历史功能整合到主界面，实现真正的多对话管理体验，用户可以在历史对话中无缝继续聊天。新增完整的人工支持工作流，当AI无法满足用户需求时可请求人工技术专家协助。

技术栈

Django HTMX Alpine.js Tailwind CSS Supabase 阿里云百炼

1. 统一主界面 全新设计

这是应用的核心界面，整合了聊天功能和历史对话管理。通过可收缩的左侧导航栏，用户可以在多个对话线程间轻松切换，并且能够在任何历史对话中继续提问。

展开状态 - 完整对话管理
收缩状态 - 专注聊天体验

核心功能特性：

  • 可收缩/展开的左侧导航栏，支持平滑动画过渡
  • "新建聊天"按钮快速创建新的对话线程
  • 历史对话列表按时间倒序排列，当前对话高亮显示
  • 用户可以在任何历史对话中继续提问和互动
  • 收缩状态下仅显示图标，节省屏幕空间
  • 汉堡菜单控制导航栏的显示/隐藏
  • 保持原有的聊天界面体验（流式输出、点赞/点踩等）

交互改进：

  • 展开状态：280px 宽度，显示完整对话标题和时间
  • 收缩状态：60px 宽度，仅显示圆点指示器
  • 点击对话项立即切换到该对话，内容淡入淡出过渡
  • 输入框占位符显示"继续在此对话中提问..."
  • 对话标题区显示"可继续对话"提示

2. 登录/注册模态框

当游客尝试使用高级功能或达到使用限制时，会弹出此模态框引导用户注册。设计简洁明了，支持快速切换登录和注册视图。

交互特性：

  • 模态框覆盖主界面，背景半透明遮罩
  • 登录和注册视图可无缝切换
  • 邮箱和密码输入框带占位符提示
  • 实时表单验证和错误提示
  • 右上角关闭按钮可关闭模态框
  • 注册需要邮箱验证激活

3. 点踩反馈弹窗

当用户对AI回答不满意并点击"👎"按钮时弹出的反馈选择界面。提供多种预设选项，其中"需要人工支持"是进入人工服务的重要入口。

交互特性：

  • 点击点踩按钮即时弹出，提供快速反馈通道
  • 四种预设反馈原因：不准确、无关、过时、需要人工支持
  • 单选设计，避免用户选择困难
  • 选择"需要人工支持"后自动进入人工支持申请流程
  • 简洁的弹窗设计，不干扰主界面

4. 人工支持请求表单

用户选择"需要人工支持"后显示的详细申请表单。收集必要的问题信息，同时自动包含相关的AI对话上下文，帮助技术专家更好地理解问题。

表单功能：

  • 问题类型分类：技术问题、产品咨询、故障排除、其他
  • 详细描述文本框，支持用户描述具体问题
  • 联系邮箱预填当前用户邮箱，可手动修改
  • 自动包含相关AI对话上下文
  • 可选择是否包含完整对话历史

5. 人工支持确认页面

用户提交人工支持请求后的确认界面。提供明确的成功反馈，并告知用户后续流程和预期时间。

确认信息：

  • 绿色勾选图标提供清晰的成功视觉反馈
  • 生成唯一工单编号，便于后续跟踪
  • 明确告知预计回复时间（24小时内）
  • 确认邮件通知已发送到用户邮箱
  • 简洁的确认界面，提升用户信心

6. 管理员后台 - 用户管理

管理员后台的用户管理模块，用于查看和管理所有注册用户。采用传统的侧边栏 + 主内容区布局，功能清晰直观。

管理功能：

  • 深色侧边栏导航，视觉层次清晰
  • 用户统计卡片显示关键指标
  • 用户列表表格展示详细信息
  • 用户状态标识（正常/受限/禁用）
  • 支持按邮箱搜索用户
  • 用户操作按钮（查看/禁用/解封）
  • 分页导航支持大量数据浏览

7. 管理员后台 - 人工支持工单 新增

人工支持工单管理界面，是Phase 1新增的核心功能。技术专家可以在此处理用户的人工支持请求，查看完整的问题描述和AI对话上下文，并直接回复用户。

工单管理功能：

  • 工单状态统计卡片：待处理、处理中、已完成、已关闭
  • 多维度筛选：按状态、时间范围筛选工单
  • 工单列表展示：ID、用户邮箱、问题类型、提交时间、状态
  • 工单详情预览：在下方区域显示完整的工单信息
  • AI对话上下文：显示用户的原始问题和AI回答
  • 在线回复编辑器：技术专家可直接回复用户问题
  • 工单操作：发送回复、更改状态、关闭工单等

工作流程：

  • 用户点踩AI回答并申请人工支持
  • 系统自动创建工单并发送邮件确认
  • 管理员后台显示新工单，按时间排序
  • 技术专家查看工单详情和AI对话上下文
  • 技术专家撰写详细的解决方案并回复
  • 系统发送回复邮件给用户
  • 工单状态更新为已完成或已关闭

8. 问答排行榜 新增

问答排行榜功能展示最受欢迎的问题和答案，基于点赞数、回答质量、问题频率等多维度权重计算。用户可以通过排行榜发现热门问题，获取高质量的技术解答，同时为产品优化提供数据支撑。

排行榜功能：

  • 多维度热度计算：综合考虑点赞数、回答次数、用户关注度
  • 分类展示：本周热门、全部时间、我的关注
  • 排行榜列表：显示问题标题、描述、统计数据、排名
  • 详细统计：点赞数、评论数、热度百分比、发布时间
  • 金银铜牌标识：前三名采用不同颜色的排名徽章
  • 查看详情：点击问题可查看完整内容和AI回答
  • 快速提问：浮动按钮支持基于热门问题发起新提问

用户交互：

  • 筛选选项卡：用户可以切换不同的时间维度查看热门问题
  • 分页导航：支持浏览更多历史热门问题
  • 问题互动：用户可以对排行榜中的问题进行点赞
  • 相关提问：基于热门问题发起新的相关提问
  • 左侧导航：通过专用入口快速访问排行榜页面

数据价值：

  • 用户需求洞察：识别用户最关心的技术问题领域
  • 产品优化指导：为产品团队提供用户痛点数据
  • 知识库建设：热门问题作为官方文档的重要补充
  • 社区活跃度：提升用户参与度和平台粘性

设计原则与规范

所有原型设计遵循以下核心原则，确保用户体验的一致性和可用性，特别针对多对话管理场景进行了优化。

色彩规范：

  • 主色调：#3b82f6（品牌蓝）
  • 成功色：#10b981（绿色）
  • 警告色：#f59e0b（橙色）
  • 危险色：#dc2626（红色）
  • 中性色：#64748b（灰色系）

新增交互规范：

  • 左侧导航栏收缩/展开动画时长：300ms
  • 对话切换时内容区淡入淡出效果
  • 悬浮时对话列表项显示操作按钮（删除、重命名等）
  • 当前选中对话项始终保持高亮状态
  • 收缩状态下的圆点指示器支持悬浮提示

响应式设计：

  • 桌面端（>1024px）：默认展开导航栏
  • 平板端（768-1024px）：默认收缩，支持展开
  • 移动端（<768px）：完全隐藏，通过汉堡菜单唤出
  • 所有尺寸下都保持核心聊天功能可用
